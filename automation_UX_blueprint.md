# Automation UX Blueprint
**Focus:** B2B Home & Commercial Infrastructure Automation
**Sources Analyzed:** Savant, Crestron, Loxone, Josh.ai

## 1. Architectural Philosophy (B2B vs B2C)
Consumer smart home sites focus on "lifestyle" (families laughing in modern living rooms). Enterprise/B2B automation sites (Loxone, Crestron) emphasize **System Architecture, Compliance, Scalability, and Integration.** 
The end-user is secondary; the primary persona is the **Integrator, Architect, Developer, or Electrical Planner.**

## 2. Core Layout Archetypes

### A. The "Command & Control" Hero
- **Observation:** None of these sites use standard SaaS imagery. Crestron uses massive architectural structures; Savant uses breaker panels. 
- **B2B Pattern:** Headline focuses on "Control Systems" or "Energy Management." 
- **Actionable Takeaway:** Module One's Hero must focus on the *network backbone*. No gadgets. Pure infrastructure and local control emphasis.

### B. The Infrastructure Grid (The Loxone Model)
- **Observation:** Loxone divides their offering into a strict module grid: *Energy, Audio, Lighting, Access, Shading, Climate, Security.* 
- **B2B Pattern:** Present the system not as "smart features" but as independent utility pillars that talk to a central brain (Miniserver/Core).
- **Actionable Takeaway:** Replace our generic "features" with a severe, icon-driven 6-column grid mapping out the exact infrastructural verticals Module One handles.

### C. The Integrator Ecosystem ("Works With" / Architect Pitch)
- **Observation:** Josh.ai and Savant dedicate massive real estate to their Dealer Portals and Interoperability layers. 
- **B2B Pattern:** "We play nice with others." The primary fear of developers is vendor lock-in that breaks when APIs change.
- **Actionable Takeaway:** Include a dedicated "Open Architecture" or "Ecosystem" section that visually lists supported local protocols (Z-Wave, KNX, MQTT, Ubiquiti, pfSense).

### D. The Resilience Argument
- **Observation:** Savant sells "Savant Power" (what happens when the grid goes down). Crestron sells "Security." Josh sells "Local AI privacy."
- **B2B Pattern:** Emphasize zero reliance on the cloud. "Local Execution" and "Offline Reliability" are the highest value propositions.
- **Actionable Takeaway:** The "Philosophy" section should be rewritten as a "Resilience/Security Protocol" block.

## 3. Applying the "Brutalist Signal" Theme to B2B
Using the existing Preset C styling:
- **Typography:** Space Grotesk and Space Mono heavily reinforce the "Developer/Integrator" demographic. 
- **Visuals:** Hard, heavy bordered grids representing server racks and breaker boxes. 
- **Interaction:** Clicks shouldn't feel frictionless; they should feel mechanical, like throwing a lever on an industrial panel.

## 4. Proposed Content Restructure outline
1. **Hero:** "Control The Infrastructure."
2. **The 6-Pillar Subsystem Grid:** (Networking, Power/Energy, Environmental, Security, Audiovisual, Surveillance).
3. **Local Execution (Resilience Module):** A dark, terminal-style section highlighting 100% cloud-free uptime.
4. **The Builder/Integrator Pitch:** Dual CTAs for "Architects" and "Developers" detailing timeline deployments (Rough-in vs. Trim-out).
5. **Request Blueprint Form:** (Already integrated, but repositioned as a "Dealer/Integrator Intake" form).
